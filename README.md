## 🚨 Проблема: Необліковані податки

Компанія не враховує податок (sales tax) у вартості замовлень. Це створює серйозні ризики:
* **Втрата прибутку:** Податкова служба не дозволить уникнути оплати податків, тому компанії доведеться покривати їх самостійно.
* **Ризик банкрутства:** Оскільки це невеликий стартап, його маржа, найімовірніше, невелика. Такі незаплановані витрати можуть призвести до збитків та потенційного краху. Компанія має лише 48 годин, щоб виправити ситуацію.

## 🗺️ Технічне рішення: Геопросторовий аналіз

Маючи координати доставки, компанія повинна визначити, який саме composite sales tax застосовується до замовлення. Процес визначення податкової юрисдикції працює наступним чином:

1. **Валідація штату та визначення округу (County):** Система не шукає по штату напряму. Замість цього вона одразу використовує Shapefile з округами штату Нью-Йорк. Одним проходом ми визначаємо, чи знаходиться замовлення в межах штату (якщо координати потрапляють у межі одного з округів), і якщо так — у якому саме окрузі. Якщо замовлення не потрапляє в жоден округ, воно відхиляється.
2. **Визначення міста (City):** Далі за допомогою ще одного Shapefile з межами міст визначається, в якому саме місті знаходиться точка доставки.

## 💰 Розрахунок податку (Tax Breakdown)

На основі зібраних геоданих система формує підсумкову ставку, використовуючи такі компоненти:
* **State Rate:** Базова ставка штату (4%).
* **County Rate:** Базових 4% округу, або інший відсоток для певних округів.
* **City Rate:** Для міст зазвичай податок нульовий, але є спеціальні міста зі своїм податком, що заміняє податок округу.
* **Special Rates:** Додаткові збори, наприклад special tax (MCTMT).

## 🚀 Запуск проєкту

### Локально (через Docker)

1. Створіть файл `.env` у кореневій директорії проєкту. Скопіюйте туди шаблон із `.env.example` та заповніть необхідні змінні.
2. Запустіть проєкт за допомогою Docker Compose:
   ```bash
   docker-compose up -d
   ```
3. Відкрийте у браузері `http://localhost:3000` — там буде доступний клієнтський інтерфейс (front-end).

### Запуск на Heroku

Проєкт також розгорнуто на Heroku. Ви можете отримати до нього доступ за посиланням: 

- https://wellness-kit-frontend-759045028065.herokuapp.com/ (frontend)
